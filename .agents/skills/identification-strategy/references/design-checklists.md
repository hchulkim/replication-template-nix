# Design Checklists

## Strategy memo structure

1. Claim
2. Estimand
3. Treatment and control
4. Data needed
5. Preferred design
6. Key assumptions
7. Diagnostics and falsification tests
8. Inference and clustering
9. Main threats

## DiD

- timing is explicit
- comparison group is credible
- parallel trends is argued and test plan is clear
- staggered adoption issues are handled if relevant
- clustering matches treatment assignment

## IV

- instrument relevance is plausible and testable
- exclusion is argued, not just named
- complier interpretation is explicit
- weak-instrument plan is stated

## RDD

- running variable and cutoff are explicit
- manipulation risk is addressed
- bandwidth and polynomial choices are justified
- local nature of the estimand is acknowledged

## Event study

- omitted period is explicit
- pre-trends will be shown
- lead and lag bins are defined
- heterogeneous timing issues are handled

## Stop conditions

- treatment cannot be defined operationally
- comparison group is not credible
- estimand changes across sections without explanation
- inference plan is missing or mismatched to assignment
