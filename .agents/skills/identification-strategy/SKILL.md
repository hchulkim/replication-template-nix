# Identification Strategy

Use this skill before coding when the empirical design is still fluid, or as a review pass when the paper already makes causal claims.

## Workflow

1. State the claim in plain language.
2. Define the estimand, treatment, control, outcome, timing, and unit of observation.
3. Compare plausible designs and rank them by credibility and data fit.
4. For the preferred design, list:
   - identifying assumptions
   - diagnostics and falsification tests
   - clustering and inference plan
   - main threats and how to answer them
5. Stop the workflow if the design cannot support the intended claim.

## Output rules

- Prefer a short strategy memo plus a checklist for implementation.
- Distinguish testable implications from untestable assumptions.
- Do not jump to code until the treatment, comparison group, and estimand are explicit.
